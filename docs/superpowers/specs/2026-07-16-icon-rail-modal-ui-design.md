# 图标工具栏与中央模态窗 UI 修订设计

## 目标

将 A1 首页现有的宽管理员侧栏改为窄图标工具栏，并将“添加链接”“上传文件”“下载任务”放入页面中央的模态窗。文件工作区继续是页面主体，v0.8.2 上传、文件操作和所有后端协议保持不变。

## 布局与入口

桌面主体使用 `64px minmax(0, 1fr)` 两列布局。左侧工具栏包含三个约 `28px` 的图标按钮，分别代表添加链接、上传文件和下载任务；按钮保持至少 `44px` 点击区域，并通过 `title`、`aria-label` 和工具提示说明用途。右侧文件区保留状态条、搜索、筛选、文件行和更多菜单。

在 `900px` 以下，左侧工具栏变为页面底部的三按钮工具条，文件工作区保持完整宽度并预留底部空间。

## 中央模态窗

三个工具区块继续由服务端渲染在首页 DOM 中，但默认隐藏在模态遮罩内：

- 添加链接：最大宽度 `520px`，保留 URL、自定义文件名、保留时间、管理密码和提交按钮。
- 上传文件：最大宽度 `560px`，保留拖放、文件名、保留时间、管理密码、进度、分片数、活跃 worker、重试次数和取消按钮。
- 下载任务：最大宽度 `860px`，保留任务状态、速度、进度、移除任务和清理已停止记录。

模态窗最大高度为 `88vh`，内容区域内部滚动。任一时刻只打开一个模态窗。普通状态可通过关闭按钮、点击遮罩或 `Escape` 关闭；打开时更新触发按钮的 `aria-expanded`，关闭后将焦点还给触发按钮。

上传进行中时，上传模态标记为 busy。点击遮罩、关闭按钮或按 `Escape` 均不关闭，用户必须等待上传完成或使用现有取消按钮。该规则只保护 UI 可见性，不修改上传 API 或服务端会话。

移动端模态宽度限制为 `94vw`、最大高度 `88vh`；任务表可在模态内部横向滚动，页面本身不产生横向滚动。

## 保留功能

- 文件搜索与类型筛选。
- 图片、视频和文本预览；视频同时提供普通下载。
- 二维码分享、一次性下载和续期更多菜单。
- 刷新、主题切换和 `/downloads/` 导航。
- v0.8.2 四个上传端点、并发 worker、重试、进度和取消。
- aria2 状态、速度、任务移除和清理停止记录。
- `/downloads/`、`/view/`、`/file/`、`/once/` 现有路由。

## 测试与验收

自动测试必须断言三个工具按钮与三个模态窗的关联 ID、ARIA 状态、关闭行为脚本、上传 busy 保护，以及原有表单和上传端点仍存在。渲染 JavaScript 必须通过 Node 语法检查，完整 Python 测试必须通过。

视觉验收覆盖 `1440x900`、`1024x768` 和 `390x844`：桌面显示窄左栏，移动端显示底部工具条，模态居中且不超出视口，文件区无重叠，上传状态和任务表可完整访问。

## 非目标

不增加后端 API、数据库、数据库字段、账号、提取码、批量 ZIP 或新前端依赖。若以后需要扩充数据库字段，必须先向用户确认。
